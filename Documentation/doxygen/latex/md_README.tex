 RT-\/\+THM is a {\bfseries{real-\/time process supervision project}}. A {\bfseries{Supervisor (parent)}} process spawns and monitors multiple {\bfseries{Worker (child)}} processes that execute simulated tasks. Workers continuously publish their status using {\bfseries{shared memory}}, and the supervisor reacts in near real time when a worker becomes unstable, hangs, or crashes.

This project also includes\+:
\begin{DoxyItemize}
\item {\bfseries{Ncurses UI dashboard}} for live monitoring in the terminal
\item {\bfseries{Logging}} with timestamps
\item A basic {\bfseries{MISRA-\/style audit checker}} (Python)
\item {\bfseries{Doxygen documentation}} generation
\end{DoxyItemize}

\hypertarget{md_README_autotoc_md2}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{md_README_autotoc_md2}

\begin{DoxyItemize}
\item {\bfseries{Multi-\/process management\+:}} uses {\ttfamily \doxylink{config_8c_a4aea176d6dafe443d89b8d850830709a}{fork()}} to create worker processes.
\item {\bfseries{IPC (Inter-\/\+Process Communication)\+:}}
\begin{DoxyItemize}
\item {\bfseries{Shared Memory (System V)}} for publishing worker statistics.
\item {\bfseries{Semaphores (System V)}} to protect shared data and avoid race conditions.
\end{DoxyItemize}
\item {\bfseries{Signals\+:}} supervisor can signal workers (pause/resume strategy) using {\ttfamily SIGUSR1} / {\ttfamily SIGUSR2}.
\item {\bfseries{Healing / Auto-\/restart\+:}} supervisor detects dead workers (via {\ttfamily \doxylink{config_8c_a4aea176d6dafe443d89b8d850830709a}{waitpid()}}) and restarts them.
\item {\bfseries{Real-\/time monitoring UI\+:}} {\ttfamily ncurses} dashboard with live stats and log messages.
\item {\bfseries{Static analysis support\+:}} includes a Python checker for basic MISRA-\/like rules.
\item {\bfseries{Doxygen documentation\+:}} codebase is documented using Doxygen comments.
\end{DoxyItemize}

\hypertarget{md_README_autotoc_md20}{}\doxysection{\texorpdfstring{IPC Model (\+High Level)}{IPC Model (High Level)}}\label{md_README_autotoc_md20}
Workers write their status into a shared array in System V shared memory, commonly using a structure like\+:


\begin{DoxyItemize}
\item PID
\item health score
\item tasks completed
\item last update timestamp
\item slow/hang detection flags
\item command field (pause/resume)
\end{DoxyItemize}

The supervisor reads this data periodically to\+:
\begin{DoxyItemize}
\item display live metrics in the UI
\item detect timeouts / instability
\item restart crashed processes
\end{DoxyItemize}

To avoid race conditions, a System V semaphore is used as a {\bfseries{mutex}} around shared memory access.
