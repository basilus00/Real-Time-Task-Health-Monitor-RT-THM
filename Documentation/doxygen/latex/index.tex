\chapter{RT-\/\+THM — Real-\/\+Time Task \& Health Monitor}
\hypertarget{index}{}\label{index}\index{RT-\/THM — Real-\/Time Task \& Health Monitor@{RT-\/THM — Real-\/Time Task \& Health Monitor}}
\label{index_md_README}%
\Hypertarget{index_md_README}%
 RT-\/\+THM is a {\bfseries{real-\/time process supervision project}}. A {\bfseries{Supervisor (parent)}} process spawns and monitors multiple {\bfseries{Worker (child)}} processes that execute simulated tasks. Workers continuously publish their status using {\bfseries{POSIX shared memory}}, and the supervisor reacts in near real time when a worker becomes unstable, hangs, or crashes.

This project also includes\+:
\begin{DoxyItemize}
\item {\bfseries{Ncurses UI dashboard}} for live monitoring in the terminal
\item {\bfseries{Logging}} with timestamps
\item A {\bfseries{MISRA-\/style audit checker}} (Python) with an output report
\item {\bfseries{Doxygen documentation}} generation
\end{DoxyItemize}

\hypertarget{index_autotoc_md5}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{index_autotoc_md5}

\begin{DoxyItemize}
\item {\bfseries{Multi-\/process management\+:}} uses {\ttfamily \doxylink{config_8c_a4aea176d6dafe443d89b8d850830709a}{fork()}} to create worker processes.
\item {\bfseries{IPC (Inter-\/\+Process Communication)\+:}}
\begin{DoxyItemize}
\item {\bfseries{Shared Memory (POSIX)}} for publishing worker statistics.
\item {\bfseries{Semaphores (POSIX)}} to protect shared data and avoid race conditions.
\end{DoxyItemize}
\item {\bfseries{Signals\+:}} supervisor can signal workers (pause/resume strategy) using {\ttfamily SIGUSR1} / {\ttfamily SIGUSR2}.
\item {\bfseries{Healing / Auto-\/restart\+:}} supervisor detects dead workers (via {\ttfamily \doxylink{config_8c_a4aea176d6dafe443d89b8d850830709a}{waitpid()}}) and restarts them.
\item {\bfseries{Real-\/time monitoring UI\+:}} {\ttfamily ncurses} dashboard with live stats and log messages.
\item {\bfseries{Static analysis support\+:}} includes a Python checker for basic MISRA-\/like rules.
\item {\bfseries{Doxygen documentation\+:}} codebase is documented using Doxygen comments.
\end{DoxyItemize}

\hypertarget{index_autotoc_md25}{}\doxysection{\texorpdfstring{IPC Model (\+High Level)}{IPC Model (High Level)}}\label{index_autotoc_md25}
Workers write their status into a shared array in POSIX shared memory, commonly using a structure like\+:


\begin{DoxyItemize}
\item PID
\item health score
\item tasks completed
\item last update timestamp
\item slow/hang detection flags
\item command field (pause/resume)
\end{DoxyItemize}

The supervisor reads this data periodically to\+:
\begin{DoxyItemize}
\item display live metrics in the UI
\item detect timeouts / instability
\item restart crashed processes
\end{DoxyItemize}

To avoid race conditions, a POSIX semaphore is used as a {\bfseries{mutex}} around shared memory access.
